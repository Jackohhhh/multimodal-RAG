# 笔记整理总结：个人效率助手工具集 (notes/测试.md)

## 要点结构
1. **基础配置与路径安全**: 核心路径限制逻辑。
2. **文件管理工具**: 提供列表查看、读取、写入及全局搜索功能。
3. **天气查询工具**: 集成外部 API 获取实时气象数据。
4. **Agent 核心逻辑**: 系统提示词设计与运行流程。

---

## 正文总结

该笔记详细记录了一个基于 LangChain 框架开发的个人效率助手后端代码。其核心设计思想是将 LLM 的能力与本地文件系统安全操作及外部 API 调用相结合。

### 1. 函数详细说明

#### **_safe_path (辅助函数)**
- **输入**: 相对路径字符串。
- **输出**: 解析后的 `Path` 对象。
- **关键逻辑**: 强制检查路径是否在 `workspace/` 目录下，禁止绝对路径或通过 `..` 进行目录穿越。
- **边界情况**: 若用户传入 `/etc/passwd` 或 `../../config.json` 将抛出 `ValueError`。
- **潜在问题**: 对于跨盘符（Windows）或复杂的符号链接支持有限。

#### **list_files**
- **输入**: Glob 模式字符串（如 `**/*.md`）。
- **输出**: JSON 格式的文件路径列表。
- **关键逻辑**: 递归或按模式扫描 workspace 内的文件并按字母排序。
- **例子**: `list_files(pattern="notes/*.md")` -> 返回所有笔记文件。

#### **read_text**
- **输入**: 文件路径、最大读取字符数（默认 120,000）。
- **输出**: 文本内容字符串。
- **关键逻辑**: 使用 UTF-8 解码并忽略错误，超出长度限制时自动截断。
- **边界情况**: 读取二进制文件可能导致内容损坏或乱码。
- **例子**: `read_text(path="notes/测试.md")`

#### **write_text**
- **输入**: 路径、内容、模式（overwrite/append）。
- **输出**: 状态确认信息。
- **关键逻辑**: 自动创建父级目录，支持覆盖或追加写入。
- **潜在问题**: 并发写入时未加锁，可能导致数据竞争。
- **例子**: `write_text(path="output/log.txt", content="new entry", mode="append")`

#### **search_in_files**
- **输入**: 搜索关键词、文件模式。
- **输出**: 包含命中文件、行号及行内容的 JSON 列表。
- **关键逻辑**: 逐行扫描文件内容，最多返回 200 条匹配结果。
- **边界情况**: 关键词在超长行中时，仅截取前 300 字符。
- **例子**: `search_in_files(query="TODO", pattern="**/*.py")`

#### **get_weather_for_location**
- **输入**: 城市名称、可选的国家代码。
- **输出**: 包含温度、体感、湿度、风速等详细信息的 JSON 字符串。
- **关键逻辑**: 
    1. 调用 Open-Meteo 地理编码 API 获取坐标。
    2. 使用坐标查询实时天气数据。
- **潜在问题**: 地名歧义（如同名城市）可能需要国家代码辅助纠偏。
- **例子**: `get_weather_for_location(city="Singapore", country_code="SG")`

### 2. 总结
该系统通过 `SYSTEM_PROMPT` 确立了严格的行动规范：限制文件读写范围，防止提示注入，并要求在生成正式文档（如周报）前先构建结构。这种设计有效地平衡了 LLM 的灵活性与本地环境的安全性。
