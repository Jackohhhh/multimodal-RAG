# 个人效率助手工具代码总结

这份笔记记录了一个基于 LangChain 和 Open-Meteo API 的个人效率助手核心逻辑，主要涵盖了文件管理安全限制、文件操作工具集、实时天气查询以及 Agent 组装。

## 目录结构
1. **基础安全配置**：路径沙箱机制。
2. **文件管理工具集**：包含列表、读取、写入及搜索功能。
3. **天气查询工具**：集成第三方地理编码与天气 API。
4. **Agent 组装与交互**：构建基于 GPT-4o-mini 的自动化流程。

---

## 正文总结

### 1. 基础安全配置 (`_safe_path`)
为了确保安全性，所有文件操作都被限制在 `workspace/` 目录下。
- **关键逻辑**：使用 `pathlib` 解析路径，并校验结果是否以 `workspace/` 的绝对路径开头。
- **边界情况**：传入绝对路径（以 `/` 或 `\` 开头）或试图通过 `../` 逃逸目录时，会抛出 `ValueError`。

### 2. 文件管理工具集

#### list_files
- **输入**：`pattern` (字符串，默认 `**/*`)，支持 glob 风格。
- **输出**：JSON 格式的相对路径列表。
- **关键逻辑**：遍历 `workspace/` 目录下匹配模式的所有文件。
- **边界情况**：若无匹配项则返回空列表。
- **潜在问题**：在包含大量文件的目录中使用宽泛模式（如 `**/*`）可能导致性能问题。
- **例子**：`list_files(pattern="notes/*.md")` 返回 `["notes/测试.md"]`。

#### read_text
- **输入**：`path` (文件路径), `max_chars` (最大读取字符数，默认 120,000)。
- **输出**：文件内容字符串。
- **关键逻辑**：安全检查后读取 UTF-8 编码内容，超过长度限制则进行截断。
- **边界情况**：文件不存在时由 `_safe_path` 抛出错误；大文件会自动附加 `[TRUNCATED]`。
- **潜在问题**：非文本文件（如图片）读取可能会产生乱码（虽有 `errors="ignore"`）。
- **例子**：`read_text(path="notes/测试.md")`。

#### write_text
- **输入**：`path` (路径), `content` (内容), `mode` (模式: `overwrite` 或 `append`)。
- **输出**：包含写入字符数和路径的确认消息。
- **关键逻辑**：自动创建不存在的父目录；支持覆盖或追加。
- **边界情况**：若 `mode` 不在限定范围内，抛出错误。
- **潜在问题**：频繁的 `append` 操作在超大文件上可能有效率影响。
- **例子**：`write_text(path="output/3.md", content="Summary data", mode="overwrite")`。

#### search_in_files
- **输入**：`query` (搜索词), `pattern` (范围，默认 `**/*.md`)。
- **输出**：JSON 列表，包含文件名、行号及命中内容预览。
- **关键逻辑**：逐行扫描匹配 pattern 的文件，搜集包含关键词的行。
- **边界情况**：命中数上限为 200 条，超出则立即返回。
- **潜在问题**：对超大文件进行全局搜索可能较慢。
- **例子**：`search_in_files(query="import", pattern="*.py")`。

### 3. 天气查询工具 (`get_weather_for_location`)
- **输入**：`city` (城市名), `country_code` (可选，ISO 国家代码)。
- **输出**：详细天气的 JSON 字符串（温度、体感、湿度、风速等）。
- **关键逻辑**：
    1. 调用 Open-Meteo Geocoding API 获取经纬度。
    2. 调用 Forecast API 获取当前实时气象数据。
- **边界情况**：找不到城市时抛出 `ValueError`。
- **潜在问题**：依赖外部 API，网络连接失败或 API 调整会导致不可用。
- **例子**：`get_weather_for_location(city="Singapore", country_code="SG")`。

### 4. Agent 组装
- **逻辑**：使用 LangChain 的 `create_agent` 将上述工具打包。
- **配置**：模型固定为 `gpt-4o-mini`，设定了强约束的 System Prompt，确保 Agent 仅在 workspace 内活动并强制使用天气工具。
